# Agent Ledger

The Agent Ledger is the authoritative continuity layer for the Hardlink
Organizer project workspace.

## Purpose

The ledger records:

- confirmed project state
- goals
- unresolved work
- decisions
- work history
- exceptions

## Files

- `AGENT_LEDGER_STANDARD.md`: required format and operating rules
- `GOVERNANCE.md`: read-only governance constraints
- `TERMS.md`: required term and acronym definitions
- `CURRENT_STATE.md`: evidence-based status snapshot
- `GOALS.md`: current goals and priorities
- `OPEN_LOOPS.md`: the only entry point for new execution work
- `DECISIONS.md`: durable decision record
- `WORK_LOG.md`: chronological work entries
- `EXCEPTIONS.md`: explicitly granted exceptions or deviations
- `SNAPSHOTS/`: point-in-time ledger captures when needed

## Working rule

Use this ledger before touching the rest of the Hardlink Organizer workspace.

Repo-level ledgers outside this project are secondary and should only be used
for cross-repo governance or catalog context.
